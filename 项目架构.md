# 项目架构

以下为 TrafficCopilot 仓库的推荐工程目录划分，用于支撑“交警手势检测/跟踪/分类 + 时序意图解析 + 车端可部署输出”的完整闭环。目录设计以模块解耦、便于集成与复现实验为目标。

```text
TrafficCopilot/
├─ README.md                       # 项目对外说明：简介、特性、快速开始、结构概览
├─ docs/                           # 方案文档与过程文档（不放代码）
│  ├─ 项目背景.md                   # 选题背景、痛点、意义与目标
│  ├─ 项目规划.md                   # 周计划与里程碑、风险预案
│  └─ 分工方案.md                   # 团队分工、模块边界与交付物定义
│
├─ deliverables/                   # 最终交付材料（比赛提交包产物）
│  ├─ report_word/                 # Word 报告与可导出的 PDF 版本
│  ├─ slides_ppt/                  # PPT/答辩稿与相关素材
│  ├─ video/                       # 演示视频与字幕、配音工程文件
│  └─ submission/                  # 最终提交打包目录（按组委会要求整理）
│
├─ config/                         # 配置中心：训练/评估/推理/部署的 YAML/JSON 等
│
├─ data/                           # 数据资产（建议在本机/服务器可软链接到大盘）
│  ├─ raw/                         # 原始数据：视频/图片/原始标注（只读，不做覆盖修改）
│  ├─ annotations/                 # 标注汇总：类别表、标注 JSON、关键点缓存等
│  ├─ processed/                   # 预处理后的数据：裁剪、对齐、增强缓存、特征缓存等
│  └─ splits/                      # 数据划分：train/val/test 列表与版本记录
│
├─ test_images/                    # 轻量级独立测试样例（用于演示/快速回归）
├─ weights/                        # 模型权重与导出产物（pth/onnx/engine）
├─ logs/                           # 训练与评估日志、可视化记录（tensorboard 等）
├─ experiments/                    # 实验对比产物：消融、鲁棒性、速度测试结果与图表
│
├─ scripts/                        # 入口脚本层：训练/评估/导出/benchmark/数据处理
│
├─ demo/                           # 演示应用层：本地可视化 Demo（如 Gradio）
│
├─ tests/                          # 自动化测试：单元测试、集成测试、回归用例
│
└─ core/                           # 核心能力库（算法与工程实现的主目录）
   ├─ model/                       # 视觉模型：骨干网络、注意力模块、融合结构
   ├─ perception/                  # 感知辅助：人体关键点/骨架分支、预处理与对齐
   ├─ intent/                      # 意图层：时序融合、FSM/窗口滤波、指令稳定化
   ├─ deploy/                      # 部署适配：ONNX/TensorRT、量化/剪枝、推理封装
   ├─ interfaces/                  # 对外接口与协议：与车端/仿真平台的对接约定
   │  └─ ros2/                     # ROS2 消息/话题约定与集成说明（如 msg 设计）
   └─ utils/                       # 通用工具：数据增强、指标计算、可视化、日志等
```
