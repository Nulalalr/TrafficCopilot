## 3. 可插拔系统：统一的“组件化”架构

系统拆成 4 类可插拔组件，通过 YAML 配置装配：

- DatasetProvider：负责加载 split 列表与类别名（当前默认 Roboflow `_classes.csv` 格式）
- Predictor：负责推理（PyTorch / ONNX / 未来可扩展 TensorRT）
- PoseOverlayProvider：负责骨架可视化（可选，不影响主推理）
- IntentEngine：负责时序稳定化输出（可选，可随时替换为更强的 FSM/Transformer）

统一装配入口：

- 系统构建：`core/system/factory.py::build_system`
- Web 入口：`web/app.py::create_app`
- 兼容入口：`demo/app.py`（直接加载 `web/app.py`）

## 4. 配置即插拔：`config/web.yaml`

### 4.1 当前默认配置（PyTorch FP16）

默认 Predictor 使用：

- `core.plugins.pytorch_classifier:PyTorchMobileNetV3Classifier`
- checkpoint：`experiments/mobilenetv3_albu_weather/best_model_quantized.pth`

PoseOverlay 使用：

- `core.plugins.mediapipe_pose_overlay:MediaPipePoseOverlay`

IntentEngine 使用：

- `core.plugins.temporal_intent:TemporalIntentEngine`

### 4.2 切换为 ONNX/INT8（示例）

将 `system.predictor` 修改为：

```yaml
system:
  predictor:
    target: core.plugins.onnx_classifier:OnnxGestureClassifier
    params:
      model_path: weights/mobilenetv3_baseline_int8.onnx
      config_path: experiments/mobilenetv3_baseline/config.yaml
      model_name: ONNX INT8 (dynamic)
      providers:
        - CPUExecutionProvider
```

说明：
- 组件实现会自动从 dataset 传入 `class_names`，并使用 `project_root` 解析相对路径
- 未来如要接 TensorRT，只需新增一个 `core/plugins/tensorrt_classifier.py` 并改 YAML 指向即可

## 5. “在线升级”的推荐方式（安全可控）

不建议在服务端提供“上传任意 Python 代码并执行”的能力（存在严重安全风险）。推荐的在线升级方式是：

1. 将新模块代码作为新文件加入仓库（例如 `core/plugins/new_model.py`）
2. 修改 `config/web.yaml` 的 `system.*.target` 指向新模块
3. 设置环境变量 `TRAFFICCOPILOT_ADMIN_TOKEN`
4. 调用 `POST /api/admin/reload`（带 `X-Admin-Token`）实现无重启加载新模块

## 6. 对外接口（Web API）

- `GET /`：Web 页面
- `POST /api/predict/upload`：上传图片推理（返回骨架叠加图 + 预测 + 指令）
- `POST /api/predict/sample`：对数据集中样本推理（支持 session 时序）
- `POST /api/session/reset`：重置 session 的时序状态
- `GET /api/model-info`：模型信息与（若配置）验证/测试准确率
- `GET /api/evaluate/<split>`：评测接口（返回 accuracy/precision/recall/f1 + 详细指标）
- `GET /api/modules`：当前加载的模块配置
- `POST /api/admin/reload`：热重载（需 Token）
