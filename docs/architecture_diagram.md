# TrafficCopilot 系统整体架构设计

## 一、架构设计原则

TrafficCopilot 的系统架构遵循以下五项核心原则，确保系统在满足功能需求的同时具备良好的工程可维护性：

| 原则 | 说明 |
| :--- | :--- |
| **模块解耦** | 各核心模块通过标准化数据接口通信，可独立开发、测试与替换 |
| **数据驱动闭环** | 从传感器输入到车辆控制指令形成完整数据闭环，不留功能断点 |
| **轻量化优先** | 以车规级边缘计算平台（如 Jetson Orin）为部署硬约束，全链路追求低延迟 |
| **安全性内置** | 在每个关键环节设计置信度校验与降级策略，避免"静默错误"影响行车安全 |
| **可复现可评估** | 所有模块的输出均支持日志记录与回放，评估指标覆盖精度、鲁棒性、实时性三个维度 |

## 三、模块详细设计（微观层）

### 3.1 传感器输入模块

| 属性 | 规格 |
| :--- | :--- |
| **输入源** | 车载前视/环视 RGB 摄像头 |
| **分辨率** | 1920×1080（典型），自适应缩放 |
| **帧率** | 25~30 FPS |
| **格式** | BGR / RGB，通过配置指定 |
| **接口** | OpenCV `VideoCapture` / ROS2 `Image` 话题 / 视频文件回放 |

**关键设计决策**：
- 支持**实时流和离线回放**两种模式共用同一流水线，便于开发调试与批量评估
- 通过配置文件切换输入源，无需修改代码

### 3.3 感知层（Perception Layer）

感知层是系统的核心视觉理解模块，负责从图像中定位交警并提取结构化特征，为后续分类与意图解析提供有效输入。该层包含**两条并行分支**，分别处理空间定位与结构先验提取。

#### 3.3.1 人体/交警检测与跟踪分支

```text
┌─────────────────────────────────────────────────────────────────────┐
│                  人体检测与跟踪分支 (Detection & Tracking)             │
│                                                                     │
│  预处理帧 ──▶ ┌──────────────┐    ┌──────────────┐                  │
│              │  目标检测     │───▶│ ROI 裁剪对齐  │                  │
│              │ (YOLO/轻量)   │    │ (扩大 1.2×)  │                  │
│              └──────────────┘    └──────┬───────┘                  │
│                                         │                          │
│                                         ▼                          │
│                                  ┌──────────────┐                  │
│                                  │ 多目标跟踪   │                  │
│                                  │ (IoU/Kalman) │                  │
│                                  └──────┬───────┘                  │
│                                         │                          │
│                                         ▼                          │
│  输出：BBox[x,y,w,h] + TrackID + DetectionScore                    │
└─────────────────────────────────────────────────────────────────────┘
```

| 子模块 | 方法 | 输出 |
| :--- | :--- | :--- |
| 目标检测 | 轻量级检测器（YOLOv8-nano / NanoDet） | 人体边界框 + 类别置信度 |
| ROI 对齐 | 边界框外扩 1.2 倍后裁剪 | 标准化的上半身区域 |
| 多目标跟踪 | IoU 匹配 + Kalman 滤波 | 跨帧一致的 TrackID |

**多目标处理策略**：
- 画面中可能存在多名交警或多个行人
- 系统根据检测置信度 + 位置/尺寸启发式筛选**最可能的交警目标**（如位于画面中央、穿着反光衣、手臂伸展）
- 其余目标保留 TrackID 但不进入手势分类流水线，减少计算开销

### 3.4 手势识别模型（轻量级视觉模型）

本模块是系统的算法核心，采用**双流特征融合架构**，支持纯图像模式与图像+骨架融合模式两种推理路径。

```text
┌─────────────────────────────────────────────────────────────────────────────────┐

│                                                                                 │
│                      ROI 图像                          骨架特征向量              │
│                     (224×224×3)                         (22-dim)                │
│                         │                                  │                    │
│                         ▼                                  ▼                    │
│           ┌─────────────────────────┐       ┌─────────────────────────┐        │
│           │   RGB 特征提取分支       │       │   骨架特征编码分支        │        │
│           │                         │       │                         │        │
│           │  MobileNetV3-Small      │       │  MLP Encoder            │        │
│           │  (或 EfficientNet-B0)    │       │  (Linear → BN → ReLU)×2 │        │
│           │                         │       │                         │        │
│           │  ┌───────────────────┐  │       │  输出：128-dim 特征向量   │        │
│           │  │ Coordinate Attn  │  │       └───────────┬─────────────┘        │
│           │  │ (CA Block)       │  │                   │                      │
│           │  │ 插入 stage3/4    │  │                   │                      │
│           │  └───────────────────┘  │                   │                      │
│           │                         │                   │                      │
│           │  输出：512-dim 特征向量   │                   │                      │
│           └───────────┬─────────────┘                   │                      │
│                       │                                 │                      │
│                       └───────────────┬─────────────────┘                      │
│                                       │                                        │
│                                       ▼                                        │
│                           ┌─────────────────────┐                              │
│                           │   特征融合 (Concat)   │                              │
│                           │   512 + 128 = 640    │                              │
│                           └──────────┬──────────┘                              │
│                                      │                                         │
│                                      ▼                                         │
│                           ┌─────────────────────┐                              │
│                           │   Dropout (0.3)     │                              │

│                                   turn_right, lane_change, unknown]             │
└─────────────────────────────────────────────────────────────────────────────────┘
```

#### 模型规格与路由逻辑

| 配置项 | 规格 |
| :--- | :--- |
| 骨干网络 | MobileNetV3-Small（3.2M 参数量）或 EfficientNet-B0 |
| 注意力机制 | 坐标注意力（Coordinate Attention），插入 stage3、stage4 之后 |
| 输入尺寸 | RGB: 224×224×3；骨架: 22 维特征向量（11 点 × 2 坐标） |
| 输出类别 | `stop`、`go_straight`、`turn_left`、`turn_right`、`lane_change`、`unknown` |
| 融合模式 | 可选：纯图像模式 / 双流融合模式（通过配置文件切换） |

**骨架可用性路由**：
- 当骨架提取置信度 > 阈值（默认 0.6）时，启用双流融合模式
- 当骨架提取失败或置信度过低（遮挡/夜间/超出画面）时，自动退化为纯图像模式，确保系统鲁棒性

### 3.5 时序意图解析模块（Intent Decoder）

该模块是系统区别于单纯"手势分类器"的关键创新点。它将逐帧的类别预测转化为具有**时序一致性**和**安全冗余**的驾驶意图。

```text
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         时序意图解析模块 (Intent Decoder)                          │
│                                                                                 │
│   逐帧预测序列                                                                    │
│   [stop, stop, straight, straight, straight, straight, stop, ...]               │
│       │                                                                         │
│       ▼                                                                         │
│  ┌─────────────────────────────────────────────────────────────┐               │
│  │                  第一阶段：滑动窗口滤波                        │               │
│  │                                                             │               │
│  │  · 窗口大小 W = 10 帧 (约 0.33s @ 30FPS)                     │               │
│  │  · 步长 S = 1 帧 (逐帧滑动)                                  │               │
│  │  · 滤波策略：窗口内多数投票 (Majority Vote)                   │               │
│  │  · 输出：平滑后的逐帧标签序列                                 │               │
│  │                                                             │               │
│  │  raw:    [A, A, A, B, A, A, B, B, B, B]                    │               │
│  │  smooth: [A, A, A, A, A, A, B, B, B, B]                    │               │
│  └─────────────────────────────┬───────────────────────────────┘               │
│                                │                                                │
│                                ▼                                                │
│  ┌─────────────────────────────────────────────────────────────┐               │
│  │                  第二阶段：有限状态机 (FSM)                       │               │
│  │                                                             │               │
│  │                           ┌─────────┐                       │               │
│  │                           │  UNKNOWN │                       │               │
│  │                           └────┬────┘                       │               │
│  │                                │                             │               │
│  │              ┌─────────────────┼───────────────────┐         │               │
│  │              │                 │                   │         │               │
│  │              ▼                 ▼                   ▼         │               │
│  │        ┌──────────┐     ┌──────────┐       ┌──────────┐     │               │
│  │        │  STOP    │     │  GO_     │       │  TURN    │     │               │
│  │        │          │     │  STRAIGHT│       │  LEFT/   │     │               │
│  │        └────┬─────┘     └────┬─────┘       │  RIGHT   │     │               │
│  │             │               │               └────┬─────┘     │               │
│  │             └───────────────┼────────────────────┘           │               │
│  │                             │                                │               │
│  │                ┌────────────┴────────────┐                  │               │
│  │                │   LANE_CHANGE           │                  │               │
│  │                └─────────────────────────┘                  │               │
│  │                                                             │               │
│  │  状态转移规则（关键安全设计）：                                │               │
│  │  · 任何状态 → STOP：连续 K_stop=3 帧预测为 stop 时触发       │               │
│  │  · 任何状态 → GO_STRAIGHT：连续 K_go=5 帧预测为 go_straight │               │
│  │  · STOP → TURN_LEFT/RIGHT：必须经过 GO_STRAIGHT 中间态       │               │
│  │  · 低频类别 (< 30% 窗口占比) → 保持当前状态，抑制抖动          │               │
│  └─────────────────────────────┬───────────────────────────────┘               │
│                                │                                                │
│                                ▼                                                │
│  ┌─────────────────────────────────────────────────────────────┐               │
│  │                  第三阶段：指令稳定化输出                        │               │
│  │                                                             │               │
│  │  · 输出附带 指令置信度 (Command Confidence)                   │               │
│  │  · 输出附带 状态持续时间 (State Duration)                     │               │
│  │  · 短期冲突检测：连续 3 帧输出不一致 → 触发 UNKNOWN            │               │
│  │  · 长期 UNKNOWN (>2s) → 触发 SAFETY_FALLBACK                │               │
│  └─────────────────────────────────────────────────────────────┘               │
│                                                                                 │
│  最终输出：{command, confidence, state_duration_ms, status}                      │
└─────────────────────────────────────────────────────────────────────────────────┘
```

#### FSM 状态转移表

| 当前状态 | 触发条件 | 下一状态 | 安全约束 |
| :--- | :--- | :--- | :--- |
| ANY | 连续 K_stop=3 帧 stop | STOP | 最高优先级，立即响应 |
| ANY | 连续 K_go=5 帧 go_straight | GO_STRAIGHT | 需确认窗口，防止误触发 |
| STOP | 连续 K_go=5 帧 go_straight | GO_STRAIGHT | 允许 |
| STOP | 连续 K_turn=5 帧 turn_left/right | GO_STRAIGHT → TURN | 必须经过中间态 |
| GO_STRAIGHT | 连续 K_turn=5 帧 turn_left/right | TURN_LEFT / TURN_RIGHT | 允许 |
| ANY | 连续 3 帧输出不一致 | UNKNOWN | 短期冲突检测 |
| UNKNOWN | 持续 > 2 秒 | SAFETY_FALLBACK | 触发安全降级 |

## 六、安全与降级策略

系统在以下环节内建了安全机制，以防止感知错误传导为车辆危险行为：

| 位置 | 风险 | 策略 |
| :--- | :--- | :--- |
| 预处理层 | 图像质量过差（过曝/全黑/严重模糊） | `quality_flag=BAD` → 下游模块不执行推理，直接输出 `SAFETY_FALLBACK` |
| 检测层 | 画面中无交警目标 | `NO_TARGET` 状态，输出 `UNKNOWN` |
| 检测层 | 多人检测到但 ID 跳变 | 基于 TrackID 连续性做目标重识别，短时丢失用 Kalman 预测 |
| 骨架层 | 关键点置信度过低 | 自动降级为纯图像模式 |
| 手势模型 | Top-1 置信度 < 阈值 | 输出附带 `LOW_CONFIDENCE` 标记 |
| 意图层 | 连续多帧输出不一致 | 触发 `UNKNOWN`，抑制指令输出 |
| 意图层 | UNKNOWN 持续 > 2s | 触发 `SAFETY_FALLBACK`，通知车辆接管 |
| 通信层 | ROS2 节点断连 | 心跳检测 + 自动重连 |
