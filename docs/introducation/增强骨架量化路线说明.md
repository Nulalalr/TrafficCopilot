# 增强、骨架与量化路线说明

## 1. 数据增强

当前训练脚本已经支持两种增强后端：

- `torchvision`
- `albumentations`

默认基线配置使用：

- `config/mobilenetv3_baseline.yaml`

天气增强配置使用：

- `config/mobilenetv3_albu_weather.yaml`

启动方式：

```bash
python scripts/train_mobilenetv3.py --config config/mobilenetv3_albu_weather.yaml
```

这份配置重点加入：

- 雨：`RandomRain`
- 雾：`RandomFog`
- 雪：`RandomSnow`
- 模糊：`MotionBlur` / `GaussianBlur`
- 压缩退化：`ImageCompression`
- 遮挡：`CoarseDropout`

## 2. MediaPipe Pose 骨架提取

脚本：

- `scripts/extract_pose_keypoints.py`

用途：

- 对 `train / valid / test` 全量图片离线提取关键点
- 输出 JSON，供后续双流融合使用

启动方式：

```bash
pip install mediapipe
python scripts/extract_pose_keypoints.py --config config/mobilenetv3_baseline.yaml
```

如果你的 `mediapipe` 是新版 `Tasks API` 风格，还需要下载一个 `.task` 模型文件，默认放到：

```text
weights/pose_landmarker_lite.task
```

可用文件名与默认脚本参数对应；也可以通过 `--model-path` 指定其他位置。

输出目录默认：

```text
output/pose_keypoints/
├── train_pose_keypoints.json
├── valid_pose_keypoints.json
├── test_pose_keypoints.json
└── summary.json
```

建议先验证：

- 检测成功率
- 侧身、遮挡、远距离样本的关键点质量

## 3. 模型压缩 / 量化

当前先搭建框架，优先做：

1. `PyTorch -> ONNX`
2. `ONNX 动态量化`

### ONNX 导出

```bash
python scripts/export_onnx.py --checkpoint experiments/mobilenetv3_baseline/best_model.pth
```

### ONNX 动态量化

```bash
pip install onnxruntime onnxruntime-tools
python scripts/quantize_onnx.py --input weights/mobilenetv3_baseline.onnx --mode dynamic
```

## 4. 建议推进顺序

1. 固化当前 baseline 指标
2. 跑增强版对比实验
3. 跑骨架提取并检查成功率
4. 再做双流融合
5. 最后做量化对比
